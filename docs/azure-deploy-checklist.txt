Azure Deploy Checklist

1. After each new deploy, run migrations:
cd /home/site/wwwroot
php artisan migrate

2. If APP_URL, OAuth, Stripe, or other env vars changed, clear config:
cd /home/site/wwwroot
php artisan config:clear

3. APP_URL must match the real Azure domain exactly.

4. For Laravel routes on Azure Linux, use this Startup Command:
cp /home/site/wwwroot/default /etc/nginx/sites-available/default && service nginx reload

5. The project must include the `default` file with the nginx rewrite to public/index.php.

6. If the site opens but styles are broken, check:
- whether assets are loading over http instead of https
- whether the browser is blocking Mixed Content

7. If the catalog is empty, check:
- whether migrations were applied
- whether products exist in the database
- whether active products exist

8. For Google login on Azure, set:
- GOOGLE_CLIENT_ID
- GOOGLE_CLIENT_SECRET
- GOOGLE_REDIRECT_URI=https://your-domain/auth/google/callback
- and add the same redirect URI in Google Cloud Console

9. For Stripe on Azure, set:
- STRIPE_KEY
- STRIPE_SECRET
- STRIPE_WEBHOOK_SECRET if webhook is used

10. If routes return 404 after deploy, check:
- whether the Startup Command was applied
- whether the App Service restarted
- whether the route works without index.php
